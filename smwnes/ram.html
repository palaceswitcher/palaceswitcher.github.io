RAM Map (2024-1-29)

Address Length Type Description Details
$0000 1 byte Mirror PPU control mirror, responsible for things like sprite size, NMI generation, etc.
$0001 1 byte Mirror PPU mask mirror, responsible for color effects and the rendering of sprites and backgrounds.
$0002 1 byte Mirror X scroll position of the PPU, copied to $2006 first.
$0003 1 byte Mirror Y scroll position of the PPU, copied to $2006 after $0002.
$0004 1 byte Misc. Unknown, possibly related to collision.
$0006 1 byte Counter Frame counter.
$0007 1 byte Misc. Slightly delayed mirror of $0006. Exact purpose is currently unknown, but it's used to continuously increment $00E4 while it's desynchronized. May be tied to the overworld.
$0008 1 byte Misc. Backup of PRG bank 42's ID, used to swap bank 42 back into the 1st PRG slot after briefly swapping the 1st sound bank in. Always #$2A.
$0009 1 byte Misc. Backup of PRG bank 41's ID, used to swap bank 41 back into the 2nd PRG slot after briefly swapping the 2nd sound bank in. Always #$29.
$000A 2 bytes Player Player X position (16-bit, Big-Endian). High byte is their screen, low byte is the coordinate.
$000C 2 bytes Player Player Y position (16-bit, Big-Endian). High byte is their screen, low byte is the coordinate.
$000E 2 bytes Player Mirror of the player's X position. Exact purpose is unknown, but it's used for things like collision and sprite position. It's inversely copied back into the player's X position in PRG bank 61.
$0010 2 bytes Player Mirror of the player's Y position. Behavior and purpose is similar to the previous location.
$0012 1 byte Player Player sprite X position.
$0013 1 byte Player Player sprite Y position.
$0014 1 byte Player Player X speed.
$0015 1 byte Player Player Y speed.
$0017 1 byte Player Player animation frame.
$0018 1 byte Player Player animation frame length (in video frames).
$0019 1 byte Player Player movement bits.
Bit 0: Horizontal direction (0 = Right, 1 = Left)
Bit 2: Vertical direction (0 = Down, 1 = Up)
Bit 6: Sprite Direction (0 = Right, 1 = Left)
$001A 1 byte Player Player sprite attributes, sets attributes of each of the player's metasprites. Only the mirroring and priority bits are ever used, and changing the palette bits will only affect the top half of the player sprite.
$001B 2 bytes Player Player animation frame pointer.
$001D 1 byte Player Player action, tied directly to animations and affects what the player is able to do. Values
$001E 1 byte Player Previous player action, used to detect action changes between 2 frames.
$0028 1 byte Player Scaled player X speed, determines how much to offset the player's X position each frame. Its value is table based and is determined by the upper nybble of the player's x speed.
$002B 1 byte Player Scaled player Y speed, determines how much to offset the player's Y position each frame. It is scaled the same way as $0028.
$003A 1 byte Player Player's sprite width, in 8x8 tiles. Graphical only.
$003B 1 byte Player Player's sprite height, in 8x8 tiles. Graphical only.
$004A 1 byte Counter Koopa shell hit counter.
$008C 2 bytes Levels Current level screen order pointer. Used to determine how the 16x16 block screens should be laid out.
$008E 1 byte Sound Music register. Changes the music to whatever ID is written to it, assuming it's different from the current song. Values
$008F 1 byte Sound Sound effect register. Plays whatever SFX ID is written to it. Values
$0090 1 byte Sound Back-up of the music register. Used to set the song back after the invincibility music plays.
$0096 1 byte Collision ID of the tile inside the player's hitbox, seems to be based off the player's head.
$0099 1 byte Collision ID of the tile below the player's feet.
$00A3 1 byte Objects Object count.
$00A4 1 byte Objects Index of the currently running object slot. Determines which object's code is being run at a given time.
$00AE 2 bytes Sprites Game mode 0 sprite animation pointer.
$00B0 1 byte Graphics Amount of 8x8 tile rows to draw to the screen.
$00B1 1 byte Graphics Amount of 16x16 tile attribute rows being updated.
$00B2 7 bytes Sprites Game mode 0 sprites. Note that these have no animation system and are animated by cycling between sprites. Sprite IDs
$00C3 1 byte Flag Boo Buddies enabled flag.
$00C4 2 bytes Objects Boo Buddies X position (16-bit).
$00C6 2 bytes Objects Boo Buddies Y position (16-bit).
$00C9 2 bytes Objects Player attribute data pointer.
$00D0 12 bytes Levels Level screen data pointers?

00 = Outside of a level (Map, title, etc)
01 = In a level
$00DF 1 byte Misc. Event trigger. Event IDs
$00E0 1 byte Misc. Current event part, used to sequence events. More info here.
